The claim in one paragraph

The brief asks for AI that reads a photo and routes the request. We measured four years of Montréal's routing first — and the routing is not the AI problem. It is 97% reproducible with a containment test and a lookup table. So the design: AI does perception (photo/voice → asset + defect); a deterministic, citable resolver does routing, and abstains to a human when sources conflict. The model never emits a department name — so it cannot misroute by hallucination.

Reproduce every number

Two dependency-free scripts (Node ≥ 18)

node shadow/montreal-challenge-verify.mjs — re-derives every proposal figure from the live City API and reports PASS / DRIFT / FAIL. ~23s, aggregation runs server-side in the City's datastore.

node shadow/shadow-run.mjs — regenerates the live scorecard against today's data.

The point of both: a juror checks our arithmetic against the source of record without trusting a word we wrote.
